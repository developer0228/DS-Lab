Practical 6(A): Implement Bully Algorithm for Leader Election
1. Aim

To implement the Bully Algorithm for leader election in a distributed system.

2. Explanation of Practical
What is Bully Algorithm?

Bully algorithm is a distributed election algorithm used to choose a coordinator (leader) among multiple processes.

Key idea:

Each process has a unique process ID.
The process with the highest active ID becomes leader.
If the coordinator fails, another process starts election.
Higher ID processes override lower ID processes during election.

It is called “Bully” because the highest-ID process always takes control.

5. How it is Implemented
Important Steps
1. Detect Coordinator Failure

A process identifies that current leader is not responding.

2. Start Election

The process sends election message to all processes having higher IDs.

3. Response from Higher Process

If higher process is alive, it replies and starts election itself.

4. Highest Active Process Wins

Highest process ID becomes coordinator.

5. Announce New Leader

New leader sends coordinator message to all processes.

6. Working of Bully Algorithm
Processes: P1 P2 P3 P4 P5

Suppose P5 fails

P2 starts election
   ↓
Election message to P3, P4, P5
   ↓
P4 replies
   ↓
P4 becomes leader
   ↓
Coordinator message sent to all

Because higher ID process dominates lower ID processes during election.

P6 overrides election started by P2.

12. Important Technical Questions for External Examiner
Q1. What happens if highest process fails?

Next highest active process becomes leader.

Example:

P5 fails, P4 becomes leader.

Q2. How is failure detected?

By timeout or no response from coordinator.

Example:

Leader not replying to request.

Q3. Why are higher IDs preferred?

They ensure unique and consistent leader selection.

Example:

P9 preferred over P4.

Q4. What is message overhead?

Extra messages exchanged during election.

Example:

Election notifications.

Q5. Is Bully Algorithm centralized?

No, it is distributed election.

Example:

Any process can start election.

Q6. What if two processes start election simultaneously?

Higher process eventually wins.

Example:

P2 and P3 both initiate election.

Q7. Why is it efficient for small systems?

Fewer processes reduce communication overhead.

Example:

5-node cluster.

Q8. What is timeout in Bully Algorithm?

Waiting period for response from coordinator.

Example:

Leader failure detection.

Q9. Can lower process become leader?

Only if all higher processes fail.

Example:

P2 elected when P3–P5 fail.

Q10. Why is leader election important?

It ensures system coordination after failures.
