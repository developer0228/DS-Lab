Practical 5: Implement Token Ring Based Mutual Exclusion Algorithm
1. Aim
To implement Token Ring based Mutual Exclusion Algorithm for controlling access to shared resources in distributed systems.

2. Explanation of Practical
What is Token Ring Mutual Exclusion?
Token Ring Mutual Exclusion is a distributed algorithm used to ensure that only one process accesses the critical section at a time.

In this algorithm:

Processes are connected in logical ring structure.

A special message called token circulates in the ring.

Only the process holding the token can enter critical section.

This prevents simultaneous access to shared resources.

5. How it is Implemented
Important Steps
1. Create Logical Ring
Processes are connected in circular manner.

2. Generate Token
A unique token is passed among processes.

3. Token Request
Process waits for token to access critical section.

4. Enter Critical Section
Only token holder enters critical section.

5. Pass Token
After execution, token is forwarded to next process.

6. Working of Token Ring Algorithm
Process Ring
P1 → P2 → P3 → P4
↑               ↓
← ← ← Token ← ← ←

Token Holder
      ↓
Access Critical Section
      ↓
Pass Token to Next Process

11. Important Viva Questions and Answers
Q1. What is mutual exclusion?
Mutual exclusion ensures only one process accesses shared resource at a time.

Example:
Single printer access.

Q2. What is Token Ring Algorithm?
It is a token-based mutual exclusion algorithm using logical ring structure.

Example:
Processes passing token sequentially.

Q3. What is token?
Token is special permission message allowing process to enter critical section.

Example:
Process holding token accessing resource.

Q4. What is critical section?
Critical section is code segment accessing shared resource.

Example:
Updating shared database file.

Q5. Why is token important?
Without token, process cannot access critical section.

Example:
Only authorized process enters resource area.

Q6. What is logical ring?
Processes are virtually connected in circular structure.

Example:
P1 → P2 → P3 → P1.

Q7. What is race condition?
Race condition occurs when multiple processes access shared resource simultaneously.

Example:
Two users editing same file.

Q8. What is starvation?
A process continuously waiting without getting resource access.

Example:
Process never receiving token.

Q9. What happens if token is lost?
No process can enter critical section until token regenerated.

Example:
Communication failure causing token loss.

Q10. What are applications of token ring?
Used in resource sharing and distributed synchronization.

Example:
Network printer management.
