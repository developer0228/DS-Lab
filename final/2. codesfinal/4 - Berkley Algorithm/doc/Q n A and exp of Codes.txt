1. Aim

To implement Berkeley Algorithm for synchronizing clocks in distributed systems.

2. Explanation of Practical
What is Berkeley Algorithm?

Berkeley Algorithm is a clock synchronization algorithm used in distributed systems where one system acts as a master server and synchronizes time among all connected systems.

The master system:

Collects time from all systems
Calculates average time
Sends adjustment values to all systems

This ensures all clocks remain approximately synchronized.

5. How it is Implemented
Important Steps
1. Master Polls All Systems

Master server requests current time from all nodes.

2. Time Collection

Each system sends its local time to master.

3. Average Time Calculation

Master calculates average clock time excluding faulty clocks.

4. Time Adjustment

Master sends correction values to all systems.

5. Clock Synchronization

All systems update their clocks accordingly.

6. Working of Berkeley Algorithm
Master Server
      ↓
Collects Time from All Nodes
      ↓
Calculates Average Time
      ↓
Sends Time Corrections
      ↓
All Clocks Synchronized

Berkeley uses average system time while Cristian’s uses external time server.

Internal vs external synchronization.

Q2. What happens if master server fails?

Synchronization process stops because master controls coordination.

Q3. How is average time calculated?

Master computes mean of all system times excluding abnormal clocks.

Average of node times.
