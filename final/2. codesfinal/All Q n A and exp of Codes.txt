Practical 5: Implement Token Ring Based Mutual Exclusion Algorithm
1. Aim
To implement Token Ring based Mutual Exclusion Algorithm for controlling access to shared resources in distributed systems.

2. Explanation of Practical
What is Token Ring Mutual Exclusion?
Token Ring Mutual Exclusion is a distributed algorithm used to ensure that only one process accesses the critical section at a time.

In this algorithm:

Processes are connected in logical ring structure.

A special message called token circulates in the ring.

Only the process holding the token can enter critical section.

This prevents simultaneous access to shared resources.

5. How it is Implemented
Important Steps
1. Create Logical Ring
Processes are connected in circular manner.

2. Generate Token
A unique token is passed among processes.

3. Token Request
Process waits for token to access critical section.

4. Enter Critical Section
Only token holder enters critical section.

5. Pass Token
After execution, token is forwarded to next process.

Practical 6(A): Implement Bully Algorithm for Leader Election
1. Aim

To implement the Bully Algorithm for leader election in a distributed system.

2. Explanation of Practical
What is Bully Algorithm?

Bully algorithm is a distributed election algorithm used to choose a coordinator (leader) among multiple processes.

Key idea:

Each process has a unique process ID.
The process with the highest active ID becomes leader.
If the coordinator fails, another process starts election.
Higher ID processes override lower ID processes during election.

It is called “Bully” because the highest-ID process always takes control.

3. Outcome of Practical

After performing this practical, we can:

Understand leader election in distributed systems.

Elect new leader automatically.

Distributed systems
Cluster management
Cloud systems
Distributed databases
Network coordination

It helps automatically elect coordinator when current leader fails.

5. How it is Implemented
Important Steps
1. Detect Coordinator Failure

A process identifies that current leader is not responding.

2. Start Election

The process sends election message to all processes having higher IDs.

3. Response from Higher Process

If higher process is alive, it replies and starts election itself.

4. Highest Active Process Wins

Highest process ID becomes coordinator.

5. Announce New Leader

New leader sends coordinator message to all processes.

6. Working of Bully Algorithm
Processes: P1 P2 P3 P4 P5

Suppose P5 fails

P2 starts election
   ↓
Election message to P3, P4, P5
   ↓
P4 replies
   ↓
P4 becomes leader
   ↓
Coordinator message sent to all

Practical 6(B): Implement Ring Algorithm for Leader Election
1. Aim

To implement the Ring algorithm for leader election in a distributed system.

2. Explanation of Practical
What is Ring Algorithm?

Ring Algorithm is a distributed leader election method where all processes are arranged in a logical ring.

Key idea:

Each process has a unique process ID.
Election message moves in circular order.
Every process adds its ID to the election message.
Highest process ID is selected as leader.
New coordinator is announced to all processes.

This ensures orderly leader selection.
